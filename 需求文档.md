# 香港医院病床实时监控大屏系统 — 需求文档

## 1. 项目概述

本项目是一个用于实时监控香港医院病床使用情况的可视化大屏系统，包含后台管理系统和多个面向大屏的实时数据展示页面。系统通过模拟生成逼真的医院运营数据，提供病床使用率、空闲病床数、病床分布图等多维度数据可视化功能，支持大屏轮播、动态加载和交互式配置。

## 3. 数据模型

### 3.1 基础数据 Excel

文件：`hospital_bed_usage_data_2.xlsx`

**Sheet1 - 医院科室信息**：
- 字段：地区、医院id、医院名称、科室id、科室名称、病房id、病房名称、病房总床数
- 数据范围：6 家医院、8 个科室、96 间病房、2109 张床位

**Sheet2 - 床位信息**：
- 字段：病房id、床位号
- 每个病房 15-30 张床

**床位标签**：自动补充楼号（A-E）和楼层（1-10），格式 `{病房id}-{楼号}-{楼层}-{编号}`

### 3.2 模拟数据生成

**病人住院记录 CSV**：
- 字段：医院名称、科室名称、病房名称、床位号、病人姓名、病人类型、入院时间、出院时间
- 文件名格式：`{医院id}_{医院名}_病人信息.csv`

**维修记录 CSV**：
- 字段：医院名称、科室名称、病房名称、床位号、维修开始时间、维修结束时间
- 文件名格式：`{医院id}_{医院名}_维修记录.csv`

## 4. 功能需求

### 4.1 后台管理

| 功能 | 说明 |
|------|------|
| **系统总览** | 展示医院数量、总科室、总床位数、生成文件数统计卡片；各医院床位占用表；各科室占用汇总表（跨医院同名科室合并） |
| **数据生成** | 选择起止日期生成模拟数据，含进度条和日志，自动清除旧数据并预热缓存 |
| **数据查询** | 按医院、科室、关键词筛选病人和维修记录 |
| **文件管理** | 查看已生成 CSV 文件列表，单文件下载或 ZIP 打包下载 |
| **大屏配置** | 6 张大屏预览卡片；拖拽排序播放顺序；每个大屏启用/禁用开关；独立设置播放时长（3-60秒） |

### 4.2 数据生成逻辑

- **日期范围**：保障生成数据在指定月份内
- **时间精度**：所有时间精确到整半点（`:00` 或 `:30`）
- **入住时长**：4-48 小时，整小时，正态分布（均值 24h，标准差 10h）
- **占用率**：逐小时总量驱动，50%-90% 动态波动（日内波动 + 月度波动 + 随机噪声）
- **维修占比**：约 5%，时长 1-48 小时
- **姓名脱敏**：中文/英文混用，中间部分用 `*` 替换，只显示首尾
- **床位不冲突**：基于小时位图的占用标记，每床位每小时唯一状态
- **超长拆分**：超过 48 小时的连续占用自动拆分为多条记录
- **存储格式**：单医院单 CSV

### 4.3 六大屏展示

| 序号 | 名称 | 核心内容 | 加载机制 |
|------|------|----------|----------|
| 1 | 各科室病床使用率 | 各科室4天对比柱状图 + 全院24H趋势折线，Y轴50%-100% | 420ms/时，不同医院轮播 |
| 2 | 医院床位占用率 | 所有医院横向柱状图 + 全域24H趋势折线，Y轴50%-100% | 420ms/时 |
| 3 | 各科室空闲病床数 | 双轴图（左轴0-50柱状+右轴40-200折线）+ 右栏转入/转出日志流 | 420ms/时，医院轮播 |
| 4 | 病床分布图 | 2D模拟地图（8种拓扑路网）+ 建筑内床位实时状态（红/绿/黄）+ 左侧楼层列表 | 3秒切换医院 |
| 5 | 科室占用率对称双轴 | 左轴占用率0-100%柱状 + 右轴空闲床数折线 + 整点日志盘点各医院空闲科室 | 420ms/时 |
| 6 | 香港医院资源总览 | ECharts地图 + 6家医院散点定位 + 占用率列表 + 总床/占用率/空闲数 | 3秒推进1小时 |

### 4.4 大屏通用机制

- **动态逐小时加载**：屏1/2/3/5 使用 420ms 推进 1 小时，模拟 24 小时变化
- **前3天对比**：所有柱状图和折线图都展示当天、前1天、前2天、前3天的数据
- **医院轮播**：屏1/3 显示完一个医院的 24h 数据后，暂停3秒自动切换到下一家医院
- **独立预览**：支持 `/preview/1` ~ `/preview/6` 独立查看每个大屏
- **总体预览**：`/preview` 按配置顺序轮播全部启用的屏幕
- **自定义顺序**：支持 URL 参数 `/preview?order=1,3,5` 指定播放顺序

### 4.5 性能指标

- 所有数据接口响应 < 10ms（缓存命中）
- 页面首次加载 < 3s（含缓存构建）
- 生成 6 家医院整月数据 < 120s
- 内存缓存小时矩阵，避免重复磁盘读取

## 5. 缓存架构

| 缓存 | 类型 | 用途 |
|------|------|------|
| `csv_cache` | dict[hospital_id] | CSV 文件的 Pandas DataFrame 缓存 |
| `hourly_matrix_cache` | dict[hospital_id][date] | 每医院每天 24 小时床位占用位图 |
| `daily_comp_cache` | dict[hospital_id][date] | 每日对比数据（4天对比） |
| `dept_summary_cache` | dict[date] | 跨医院科室汇总逐小时数据（屏5） |
| `hospital_geo_cache` | dict[date] | 医院地理位置与占用率（屏6） |

## 8. 启动与使用

### 8.1 启动

```bash
cd mem-dashboard-v2
pip install -r requirements.txt
python3 run.py
```

访问 `http://127.0.0.1:8989`

### 8.2 使用流程

1. **管理后台** → 数据生成 → 选择日期 → 点击生成
2. 等待进度条完成（约 60s）
3. **大屏配置** → 拖拽调整顺序 → 设置时长 → 保存
4. 点击卡片预览或 **总体预览**
5. **总览** 查看全市数据统计

### 8.3 访问入口

| 页面 | URL |

## 9. 已知限制与后续规划

### 9.1 当前限制

- 所有数据为模拟生成，非真实医院数据
- 地图为简化 GeoJSON 多边形，非真实香港行政区划
- 单屏浏览器查看，未适配多屏物理拼接
- 无数据库持久化，重启后需重新生成

### 9.2 扩展方向

- 接入真实医院数据源 API
- 多屏 HDMI 拼接墙输出
- SQLite/PostgreSQL 数据持久化
- 用户权限与多用户管理
- 大屏模板自定义编辑器
- 占用率超阈值实时告警
- 历史趋势月/季度报表
- 中英文多语言支持
